从零开始构建PHP应用

  • PHP

罐头跳槽了，没来得及把工作环境压缩带走，在新公司当然要折腾一番把环境搞回来。给了一台干净得要命的笔记本，真的是从零开始，中途遇到了不少坑，特地用笔记记下来。

  1. 下载了网上最新版的VMware Workstation12
    传送门：http://sw.bos.baidu.com/sw-search-sp/software/d34ff7ffac8/VMware-workstation_full_12.1.1.6932.exe

    VMware Workstation 12序列号：
    5A02H-AU243-TZJ49-GTC7K-3C61N

  2. 接收一个君哥给给镜像CentOS-6.3-x86_64-minimal.iso
    包颇大（QQ接收都耗了30分钟），就不贴上来了，
    可到官网下载 https://www.centos.org/

  3. 安装centOs
    创建新的虚拟机->自定义->Workstation 12.0->创建的虚拟机将包含一个空白硬盘->linux centos 64->虚拟机命名和位置->自定义硬件->建立虚拟机之后再选择ISO路径，然后开始安装。
    系统盘成功引导后，出现下面的界面

    3.1 选择Install or upgrade an existing system 安装或升级现有的系统
    3.2 出现是否对CD媒体进行测试的提问，这里选择“Skip”跳过测试。
    3.3 选择语言 中文（简体）、美国英语式
    3.4 键盘模式，默认即可
    3.5 是，丢弃所有数据
    3.6 时区选择，取消“系统时钟使用UTC时间”前面的勾
    3.7 设置root密码
    3.8 选择最后一项，自定义分区
    假如硬盘总共5G
    系统文件类型：swap 大小（M）：1024
    挂载点：/ 文件系统类型：ext4 选中“使用全部可用空间”
    点格式化，将修改写入磁盘
    3.9 进入安装，安装完毕请重启
    图文传送门：http://www.dedecms.com/knowledge/servers/linux-bsd/2012/0819/8398.html

  4. 解决网络问题
    4.1 使用NAT 模式（N）：用于共享主机的IP地址
    4.2 “编辑”-“虚拟网络编辑器”选中VMnet8，点选NAT（与虚拟机共享主机IP地址），把DHCP勾上（那两个框都勾上）。
    4.3 开启宿主机（windows7）中的VMware DHCP Service 和VMware NAT Service服务。
    4.4 手工配置ip，直接编辑 vi /etc/sysconfig/network-scripts/ifcfg-eth0
    ifcfg-eth0 文件内容如下：

    只需要按实际情况修改 DEVICE HWADDR NETMASK IPADDR 这几项即可
    ipaddr可以通过寄主机获取，NAT使用的 VMware Network Adapter VMnet8 的网卡

    然后service network restart
    再ping 下试试 是不是通了呢。
    4.5 在良明同学的帮助下，配置在同一局域网网内可访问
    先把虚拟机关闭了 > 编辑 > 虚拟网络编辑器

    win10貌似要点击以下按钮，获得管理员权限

    选择VMnet8（NAT模式）->点击NAT设置

    配置一个端口转发（没有点击添加，已有可以点击属性）

    主要配置 [主机端口] [虚拟机端口] [虚拟机IP地址]

  1. 安装SSH，使用上CRT
    5.1 前话（装个git，在gitbash下使用ssh-keygen生产SSH 公钥，或者用svn也可以）
    5.2 安装SSH：
    yum install ssh
    启动SSH：
    service sshd start
    设置开机运行：
    chkconfig sshd on
    一般默认CentOS已经安装了OpenSSH，即使你是最小化安装也是如此。
    5.3 修改vi /etc/ssh/sshd_config
    Port 22 指定SSH连接的端口号，安全方面不建议使用默认22端口（个人习惯是61618）
    5.4 修改vi /root/.ssh/authrized.keys
    把SSH公钥id_rsa.pub输入进入
    5.5 CentOS防火墙的关闭，关闭其服务即可：
    查看CentOS防火墙信息：/etc/init.d/iptables status
    关闭CentOS防火墙服务：/etc/init.d/iptables stop

  2. 安装lnmp
    6.1 前话（装个wegt，yum -y install wget）
    6.2 下载一个国内LNMP 1.3稳定版
    wget https://api.sinas3.com/v1/SAE_lnmp/soft/lnmp1.3-full.tar.gz –no-check-certificate
    （包也是颇大的，400多M，下载需要耗时30min左右）
    6.3 解压进入安装
    tar zxf lnmp1.3-full.tar.gz && cd lnmp1.3-full && ./install.sh lnmp
    6.4 安装流程
    输入数据库密码->选择mysql版本（MySQL 5.5.48）->选择PHP版本（PHP 5.6.22）->选择缓存工具（Don’t install Memory Allocator.）->回车开始安装
    安装时间可能会几十分钟到几个小时不等，主要是机器的配置网速等原因会造成影响。（请耐心等候，本人装了2个小时）
    6.5 安装完成
    如果显示Nginx: OK，MySQL: OK，PHP: OK

    并且Nginx、MySQL、PHP都是running，80和3306端口都存在，并提示Install lnmp V1.3 completed! enjoy it.的话，说明已经安装成功。

  3. 修改一些常用配置
    （前言：安装一下vim可以显示代码高亮，yum -y install vim）
    把一些常用的脚本放置root（主要包含php启动、nginx重启等）
    chmod +x     增加一下执行权限（这样就可以愉快的使用了）

    7.1 PHP配置
    PHP配置文件：/usr/local/php/etc/php.ini
    我们要开启php的错误日志
    display_errors = On
    html_errors = On
    error_log = /data/logs/php_errors.log（日志目录按自己习惯）
    设置完毕之后，重启一下PHP令其生效（~/fastcgi_restart ）
    php-fpm配置文件：/usr/local/php/etc/php-fpm.conf
    listen = /tmp/php-cgi.sock
    （这个sock要注意一下）

    7.2 Nginx配置
    Nginx主配置(默认虚拟主机)文件：/usr/local/nginx/conf/nginx.conf
    添加的虚拟主机配置文件：/usr/local/nginx/conf/vhost/域名.conf

    7.3 Mysql配置
    MySQL配置文件：/etc/my.cnf

    7.4 设置 命令别名~/.bashrc 和 开机脚本/etc/rc.local

[localhost]#  vi ~/.bashrc
--------- 增加两句（个人习惯把mysql密码设置到/data/save/mysql_root,避免忘记）----
alias vi='vim'
alias con_mysql='mysql -uroot -p`cat /data/save/mysql`'
[localhost]vi /etc/rc.local
---------------------------------------------------------------------------------------------
[localhost]# source ~/.bashrc

[localhost]# vi /etc/rc.local
--------- 由于是用vm装的虚拟机，当关机/重启时希望不用手动启动/关闭某些服务--------
/etc/init.d/iptables stop
~/nginx_reload
~/mysql_start
~/fastcgi_start
---------------------------------------------------------------------------------------------
